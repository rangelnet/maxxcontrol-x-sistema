                    RESUMO RÁPIDO: MESH TV vs TV-MAXX-PRO

DATA: 1º de Março de 2026
ANÁLISE: Completa
STATUS: ✅ Pronto para Implementação

================================================================================
                              O QUE FOI DESCOBERTO
================================================================================

1. MESH TV USA LOGIN COM JWT TOKEN
   - Tela de login (ActivityMac)
   - Usuário digita credenciais
   - Backend retorna JWT token
   - App armazena em SharedPreferences
   - App usa em todas as requisições

2. TV-MAXX-PRO USA TOKEN FIXO
   - Sem tela de login
   - Token fixo no código
   - App busca config na inicialização
   - Mais rápido e simples
   - Funciona bem para TV Box único

3. AMBAS USAM MESMA ARQUITETURA
   - Painel Web (React)
   - Backend API (Node.js)
   - App Android (Compose/Kotlin)
   - Banco de Dados (PostgreSQL)

================================================================================
                              FLUXO DO MESH TV
================================================================================

1. App Inicia
   ↓
2. Mostra Tela de Login (ActivityMac)
   ↓
3. Usuário Digita Credenciais
   ↓
4. POST /auth/login
   ├─ Envia: username, password, device_id
   └─ Retorna: JWT token
   ↓
5. App Armazena Token (SharedPreferences)
   ↓
6. GET /api/channels
   ├─ Header: Authorization: Bearer {token}
   └─ Retorna: Lista de canais
   ↓
7. App Renderiza Canais
   ↓
8. Usuário Seleciona Canal
   ↓
9. App Reproduz Stream
   ↓
10. Usuário Faz Logout
    ├─ DELETE /auth/logout
    └─ App Remove Token

================================================================================
                            FLUXO DO TV-MAXX-PRO
================================================================================

1. App Inicia
   ↓
2. GET /api/app-config/config
   ├─ Header: X-Device-Token
   └─ Retorna: config
   ↓
3. GET /api/branding/active
   └─ Retorna: logo, cores, splash
   ↓
4. GET /api/iptv-server/config
   └─ Retorna: credenciais Xtream
   ↓
5. App Inicializa XtreamRepository
   ↓
6. App Renderiza Canais
   ↓
7. Usuário Seleciona Canal
   ↓
8. App Reproduz Stream

================================================================================
                              ENDPOINTS PRINCIPAIS
================================================================================

MESH TV:
  POST   /auth/login          → Autentica usuário
  DELETE /auth/logout         → Faz logout
  GET    /auth/validate       → Valida token
  GET    /api/channels        → Lista canais
  GET    /api/categories      → Lista categorias

TV-MAXX-PRO:
  GET    /api/app-config/config       → Config do app
  GET    /api/branding/active         → Branding
  GET    /api/iptv-server/config      → Credenciais Xtream
  POST   /api/mac/register-device     → Registra dispositivo
  GET    /api/clients/verify/{mac}    → Verifica autorização

================================================================================
                              SEGURANÇA
================================================================================

MESH TV (JWT Token):
  ✓ Token por usuário
  ✓ Token expira
  ✓ Múltiplos usuários
  ✓ Mais seguro
  ✗ Mais complexo

TV-MAXX-PRO (Token Fixo):
  ✓ Token fixo
  ✓ Não expira
  ✓ Um "usuário" (TV Box)
  ✓ Mais simples
  ✗ Menos seguro

                              3 OPÇÕES DE IMPLEMENTAÇÃO
================================================================================

OPÇÃO 1: MANTER ATUAL (Recomendado Agora)
  ✓ Simples
  ✓ Rápido
  ✓ Já implementado
  ✓ Funciona bem para TV Box único
  ✗ Menos seguro
  ✗ Sem múltiplos usuários

OPÇÃO 2: IMPLEMENTAR LOGIN (Para Múltiplos Usuários)
  ✓ Mais seguro
  ✓ Múltiplos usuários
  ✓ Padrão da indústria
  ✓ Melhor controle
  ✗ Mais complexo
  ✗ Mais lento
  ✗ Menos amigável para TV Box

OPÇÃO 3: HÍBRIDA (Recomendado Futuro)
  ✓ Melhor dos dois mundos
  ✓ Flexível (login ou token fixo)
  ✓ Seguro (JWT quando disponível)
  ✓ Rápido (token fixo como fallback)
  ✗ Mais complexo
  ✗ Mais código
