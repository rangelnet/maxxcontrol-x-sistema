🧪 PRÓXIMOS PASSOS - TESTAR TUDO
═══════════════════════════════════════════════════════════════════════════════

1️⃣ INICIAR O SERVIDOR BACKEND
   cd maxxcontrol-x-sistema
   npm start
   
   ✅ Servidor deve iniciar em: http://localhost:3000

2️⃣ INICIAR O PAINEL ADMIN
   cd maxxcontrol-x-sistema/web
   npm run dev
   
   ✅ Painel deve abrir em: http://localhost:5173

3️⃣ TESTAR NO PAINEL ADMIN
   ├─ Fazer login no painel
   ├─ Ir para "Dispositivos"
   ├─ Verificar coluna "IPTV Server" (deve mostrar "Não Configurado")
   ├─ Clicar no botão roxo "Test API" (ícone TestTube)
   ├─ Configurar uma URL de teste:
   │  └─ Exemplo: https://painel.masterbins.com/api/chatbot/ABC123/TOKEN456
   ├─ Salvar e verificar se aparece mensagem de sucesso
   ├─ Clicar na célula "IPTV Server"
   ├─ Configurar servidor IPTV:
   │  ├─ URL: http://exemplo.com:8080
   │  ├─ Usuário: usuario123
   │  └─ Senha: senha123
   ├─ Salvar e verificar se a coluna IPTV atualiza
   └─ Abrir painel em 2 abas e testar atualizações em tempo real

4️⃣ TESTAR BACKEND APIs (OPCIONAL)
   
   # Buscar URL de teste (substitua o MAC)
   curl http://localhost:3000/api/device/test-api-url/AA:BB:CC:DD:EE:FF
   
   # Configurar URL de teste (precisa de token JWT)
   curl -X POST http://localhost:3000/api/device/test-api-url \
     -H "Authorization: Bearer SEU_TOKEN" \
     -H "Content-Type: application/json" \
     -d '{"mac_address":"AA:BB:CC:DD:EE:FF","test_api_url":"https://exemplo.com/api"}'

5️⃣ TESTAR NO APP ANDROID
   ├─ Compilar e instalar app atualizado
   ├─ Abrir app e ir para tela de login
   ├─ Clicar em "TESTE GRÁTIS"
   └─ Verificar logs no Logcat:
      ✅ "Buscando test_api_url para MAC: ..."
      ✅ "test_api_url encontrado: ... (has_custom_url: true)"
