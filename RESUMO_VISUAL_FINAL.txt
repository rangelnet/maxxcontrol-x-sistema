🔄 FLUXO DE AUTENTICAÇÃO
================================================================================

PRIMEIRA EXECUÇÃO:
  1. App inicia → SplashScreen
  2. Sem token salvo → LoginScreen
  3. Usuário faz login (email + senha)
  4. POST /api/auth/login → Backend valida
  5. JWT token retornado
  6. Token salvo em SharedPreferences
  7. App navega para HomeScreen

PRÓXIMAS EXECUÇÕES:
  1. App inicia → SplashScreen
  2. Token encontrado em SharedPreferences
  3. GET /api/auth/validate-token → Backend valida
  4. Token válido? → HomeScreen (direto!)
  5. Token inválido? → LoginScreen (novo login)

LOGOUT:
  1. Usuário clica em Logout
  2. DELETE /api/auth/logout → Backend invalida
  3. Token removido de SharedPreferences
  4. App navega para LoginScreen

🚀 COMO USAR
================================================================================

1. COMPILAR APK
   cd TV-MAXX-PRO-Android
   ./gradlew assembleDebug

2. INSTALAR EM TV BOX
   adb connect <IP>:5555
   adb install -r app/build/outputs/apk/debug/app-debug.apk

3. TESTAR
   • Abrir app
   • Fazer login com email/senha
   • Fechar app
   • Reabrir app
   • ✅ Esperado: App navega direto para Home (sem login!)

4. MONITORAR LOGS
   adb logcat | grep -E "Auth|Login|Session"

5. VERIFICAR TOKEN
   adb shell cat /data/data/com.tvmaxx.pro/shared_prefs/tvmaxx_session.xml

================================================================================
📱 ENDPOINTS DA API
================================================================================

POST /api/auth/login
  Entrada: email, password, device_id, device_model, app_version
  Saída: token, user, config
  Status: 200 (sucesso) | 401 (erro)

GET /api/auth/validate-token
  Entrada: Authorization header com token
  Saída: valid, user, expires_in
  Status: 200 (válido) | 401 (inválido)

DELETE /api/auth/logout
  Entrada: Authorization header com token
  Saída: success, message
  Status: 200 (sucesso) | 401 (erro)
